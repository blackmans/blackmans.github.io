git学习总结

参考：http://git-scm.com/book/zh/v1 2-4章

第二章 第一节 获取git仓库

创建一个git仓库，两种方法：

第二章 第二节 记录每次更新到仓库

所有工作目录下的文件只有两种状态：已跟踪和未跟踪；
untracked未跟踪/unmodified未修改/modified已修改/staged
只要修改了文件并且没有commit操作，文件就是处于已修改状态；
（git checkout file/git reset HEAD 分别是将未跟踪和已暂存的代码变为撤销和未暂存状态）

文件的状态周期
当然我们可以随时使用git status来查看当前的状态；
创建的新的文件就处于untracked未跟踪状态，而且你不进行跟踪或者其他操作，git不会私自进行处理；
执行git add以后文件就处于了已跟踪暂存状态，git add 后面可以跟一个具体的文件，也可以跟一个文件夹（跟一个文件夹名称的时候，表示递归文件夹中的文件）；
git add 用于合并时把有冲突的文件标记为已解决状态????
git在暂存的时候，其实保存的是add之前的版本，并不是文件本身；
.gitignore 存放忽略提交的设置；文件中可以使用goble模式，也就是简化的正则；

移除文件，就是从已跟踪文件中移除或者说从暂存盘中移除；两种情况：
一：删除文件，此文件既不在暂存区也不在文件目录中；
rm 文件名
git rm 文件名
git commit
二：只是删除文件在暂存区中的快照，文件还保留在目录中，也就是说将文件变为未跟踪状态；
git rm —cached 文件名
删除、add、cimmit都可以跟一个文件路径
如：.txt或者log\.txt

文件重命名
git mv filed_from filed_to
相当于执行了
mv filed_from filed_to
git rm filed_from
git add filed_to

第三章 第三节 查看提交历史

查看修改的记录
git log 查看修改的历史记录，不加参数默认按照修改的时间，修改最近的排在最前面；
git log -p -n 使用时不行
选项 说明
-p 按补丁格式显示每个更新之间的差异。在-p后面添加 -n 表示可以查看最近的n条记录
—word-diff 按 word diff 格式显示差异。
—stat 显示每次更新的文件修改统计信息。
—shortstat 只显示 —stat 中最后的行数修改添加移除统计。
—name-only 仅在提交信息后显示已修改的文件清单。
—name-status 显示新增、修改、删除的文件清单。
—abbrev-commit 仅显示 SHA-1 的前几个字符，而非所有的 40 个字符。
—relative-date 使用较短的相对时间显示（比如，“2 weeks ago”）。
—graph 显示 ASCII 图形表示的分支合并历史。
—pretty 使用其他格式显示历史提交信息。可用的选项包括 oneline，short，full，fuller 和 format（后跟指定格式）。
—oneline —pretty=oneline —abbrev-commit 的简化用法。

选项 说明
-(n) 仅显示最近的 n 条提交
—since, —after 仅显示指定时间之后的提交。
—until, —before 仅显示指定时间之前的提交。
—author 仅显示指定作者相关的提交。
—committer 仅显示指定提交者相关的提交。

第二章 第四节 撤销操作

修改最后一次提交

使用git commit —amend 相当于修改最后一次commit的注释，并且和上次提交使用同一个快照；
git commit -m “”
git add .
git commit —amend
三步操作相当于让上一步提交与本次提交合并为一次，只有一次快照，也相当于第二次修正了前一次的内容；

取消已经暂存的文件
git reset HEAD filed 使文件变为为跟踪状态；

取消对文件的修改
当修改文件后发现之前的修改完全没有必要，这是文件还没有add操作也就是说修改后的文件还没有进入暂存区域，这时想撤销操作：
git checkout file 就可以车修修改的内容；操作以后就相当于删除了你上一步的操作，将会无法恢复，在git中只要提交的文件操作都是可以被找回来的，无法恢复的部分只是没有提交git的部分；

第二章 第五节 远程仓库的使用

查看当前仓库的远程库
git remote
在克隆之后远程的工程之后，至少可以看到origin的远程库，git默认使用origin作为你所克隆的原始库；
你可以加上-v git remote -v 显示对应的地址；
（这里指的多个远程仓库是什么意思）

添加远程仓库
git remote name url/name.git
git push -u origin master
此时远端也必须创建仓库

从远程仓库抓去数据
git fetch 远程库的名字（相当于更新已关联的远程仓库的更新文件，更新到本地可以将代码合并到当前工程的指定分支；）
相当于将远程更新的内容拉取到本地，并不会自动merge本地；
那么git fetch加上 git merge就相当于git pull；
git fetch一般可以使用在，当你和同事开发一个项目的不同部分的时候，你们可能会有不同的分支，但是你们可能会最终提交到master或者一个相对稳定的分支上，这时候如果你的同事开发过程中提交以后，你再提交之前就要fetch一下；
（当你的本地仓库拥有多个远程分支的时候，你可以切换到新的远程仓库对应的分支，也可以将这个远程分支的代码合并到你的当前分支；）

推送数据到远端
git push origin master（将本地的分支推向远端，git push remote-name branch-name）
只要确定在服务器上具有写权限和在同一时间没有其他人进行推送操作，就可以完成推送操作；
在推送之前必须确保远端没有其他人更新了新数据，否则你的推送将会驳回，所以在pull之前要进行pull拉取操作；

查看远程仓库的信息
git remote show 远程名字

远程仓库的重命名和删除
git remote rename oldname newsman (修改远程仓库在本地的名字)
git remote rm name

第二章 第六节 打标签

列出所有标签
git tag 显示的标签按照字母顺序排序；
还可以按照特定的规则搜索：
git tag -l “v1.4.2.*”表示只搜索v.1.4.2的版本标签；

新建标签（GPG签署标签）
标签一般分为两种：轻量级和含附注，一般使用含附注的
git tag -a name -m “注释说明”
可以使用查看标签的信息
git show 标签名
创建一个轻量级的标签
git tag 标签名 就可以了
即便在以前的历史提交中忘记打标签，现在也是可以在指定的提交打标签的；
我们只需要
git log查看历史提交记录
git tag -a 标签名 提交时的校验和

分享标签
默认推送到远端时，是不会将tag推送的，需要显性的将tag推送到远端；
git push origin tangle
git push origin —tags提交全部

在使用git的过程中，我们也可以给常用的命令设置别名；也可以设置新的命令；

第三章 第一节 何谓分支

git branch name
git checkout name
创建并且换分支
其实git是通过HEAD指针的方式来判断当前分支的，当HEAD指向哪一个分支时，就证明用户是在哪一个分支上工作；其实HEAD指针是随着用户的提交向前更新的，其实可以这么认为，HEAD在某种意义上就是当前分支的一个别名；当你在当前分支上提交更新多次代码以后，再切会master时，master的HEAD还是保持在切换时的状态；现在在master分支下，其实git只是起用了现在状态下的快照，这样就能在不影响其他分支的情况下，朝着另一个方向继续开发；
其实新建一个分支，仅仅是一个快找的生成，所以速度才会很快；

第三章 第二节 分支的新建和合并

git merge name 将mame分支合并到当前分支；
git branch -d name 删除分支；
当merge时提示冲突，可以使用git status来查看文件的状态，所有需要merge的文件的状态都是unmerge；
merge完以后，在使用add/commit来再次提交
git push -u origin my_branch,关联本地与远程分支；

第三章 第三节 分支的管理

直接使用git branch 是列出所有分支；
git branch -v 查看最后提交分支的信息；
可以使用git branch —merged 和 git branch —no-merged来查看已合并和未合并的分支；
（理论上已合并的分支就可以删掉，因为工作成果已经合并进来了）

第三章 第四节 利用分支进行开发的工作流程

一般master上的代码是比较稳定分，并且master分支上的代码比较老；
git分支可以帮助你很好的处理紧急的任务或者多人开发统一项目，也可以很方便的创建和删除一个分支；

第三章 第五节 远程分支

git fetch 远程在本地的名字 是将远程的代码拉取到对应的本地指针下；我们可以将这个本地指着下的代码合并到当前分支上；
跟踪分支：就是和远程的分支有直接联系的本地分支，就是直接使用git push就可以提交到远端的仓库和分支；
在clone一个远程仓库时，git会默认创建一个master的分支跟踪origin/master；
可以使用git checkout -b [分支名] [远程名][分支名];
删除一个远程分支
git push [远程库名] :[远程分支]

第三章 第六节 分支的衍合（还没大看明白）

衍合（rebase）和merge是实现一样的功能，只是实现的过程不一样，rebase就是将一分支里的改动在另一个分支里面重新执行一遍；

第四章 第一节 服务器上的git协议

git支持四种协议：本地协议、ssh协议、git协议、http(s)协议

第四章 第二节 在服务器上部署git

首先在将现有仓库到处一份裸仓库：git clone —bare name name.git
注：git clone 相当于git init 加上git fetch
将裸仓库复制到服务器上：scp
-r my_project.git
user@git.example.com:/opt/git将裸仓库my_project.git复制到服务器git.example.com下/opt/git下面；
现在所有拥有写权限的用户都可以通过clone对用的地址来拉去此工程；
账号管理可以给每个拥有写权限的用户申请一个服务器的账号，或者在sshauthorized_keys 文件内添加各自的ssh公钥；

第四章 第三节 生成ssh公钥

ssh公钥的默认目录：~/.ssh
cd ~/.ssh检查电脑中是否已经生成过ssh公钥；
ssh keygen生成公钥

本地设置hosts不能及时生效的原因

因为服务器设置了 keep-alive ！次要原因是存在浏览器 DNS 缓存和系统 DNS 缓存。

服务器设置了 keep-alive

存在浏览器 DNS 缓存和系统 DNS 缓存

工作中常用的方式

  1. 重启浏览器

因为重启浏览器后所有连接都关闭啦，当然也包括长连接；

  1. 隐私模式打开

因为隐私模式下不会复用 TCP 连接，新开连接的时候，会重新解析 DNS 域名；

  1. 修改之后，等一会儿…

“等一会儿”，要稍微等久一点，keep-alive 的默认设置是 120s，开发者也有可能增大或者减小这个配置，所以“等一会儿”也是很伤神的=。 =
