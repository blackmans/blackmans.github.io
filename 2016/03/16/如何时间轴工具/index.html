chrome调试工具之Timeline如何使用时间轴工具（翻译）

  • chrome
  • timeline
  • 翻译

使用Chrome DevTools时间轴面板来记录和分析您的应用程序运行时的所有活动，对你的应用来说它是审查感知性能问题最好的场所。

时间轴工具的概要介绍

时间轴工具有四个面板组成

1.控制面板：开始录制，停止录制，以及配置在录制过程中获取那些信息。
2.概要面板：页面性能的信息高度概括，更详细的信息在下面展示。
3.火焰图：CPU堆栈跟踪的可视化界面。在你的火焰图上你看看到一到三个点，垂直的线；蓝色的线代表DOMContentLoaded事件（作者认为：DOMContentLoade触发，即页面文档完全加载并解析完毕之后,会触发DOMContentLoaded事件），绿色代表第一次渲染完成，红色代表页面开始渲染的时间点，红色代表load事件（作者认为：触发load事件，整个页面完全加载完毕）。
4.详细面板：当一个事件被选中时，这里会显示关于此事件的详细信息，如果你没有选择具体的事件，那么这里显示的是大概的一个时间。

概要面板

概要面板由三个图形组成

1.FPS，每秒钟帧数，绿色的块越高，表示FPS越高；FPS图形上方的红色区域代表长帧，这可能是JANK（卡顿）是很好的替换。
2.CPU，CPU资源，这个表格显示了什么事件类型占用CPU资源。
3.NET，没个彩色块代表一种资源，指示条越长证明获取资源花费的时间越长。各个指示条的较亮部分表示资源下载的等待时间（请求发出到资源第一个字节开始下载，这段时间），较深的颜色部分代表开始下载到下载完成的时间。

各颜色条代表的代码如下：HTML是蓝色，javascript是黄色，CSS饰紫色，媒体资源是绿色，其他资源是灰色。

做一个纪录

为了纪录页面加载的情况，首先打开时间轴面板，打开想记录的页面并且重新刷新页面。时间轴面板自动纪录刷新页面。

纪录一个交互页面的情况，打开时间轴面板，然后按下左上角的灰色圆圈按钮，或者按下键盘快捷键command+e(Mac)，ctrl+e(windows/Linux)。在录制的过程中这个按钮变成红色，再次按下录制按钮或者按下快捷键停止本次录制。

录制过程中的提示

1.尽可能的确保录制的比较短，录制的越短普遍容易分析。
2.避免不必要的动作（操作），避免一些不必要的操作（鼠标点击，网络加载等）这些是和你的纪录和分析是无关的；例如你想记录点击登陆按钮以后发生了什么，你不应该滚动页面，加载图片等等。
3.禁止浏览器缓存，当纪录网络情况时，这是一个好主意。
4.禁止扩展，Chrome extensions can add unrelated noise to Timeline recordings of your application. Open a Chrome window in incognito mode, or create a new Chrome user profile to ensure that your environment has no extensions.

查看详细记录

当你选择火焰表中的某一个结果时，详细窗口中就会显示关于此结果的详细信息。

某些选项卡，就像Summary，展示当前所有结果类型。其他选项卡只适合某些结果类型；可以查阅每个时间轴上的结果的详细信息。

搜索记录

当你查找结果类型时，可能想查看某一个结果类型；如：查看所有Parse HTML结果类型。
按下Command+f(Mac),ctrl+f(windows/Linux)，就会出现搜索输入框；输入你想查找的结果类型，例如：Event。
此搜索栏只适合当前选定范围之内的时间内，所有时间以外的结果都不会显示在结果里面。
向上向下移动箭头可以按顺序浏览结果，第一个结果是所选时间段最早的，最后一个是所选时间段最晚的；每次向上向下移动，都可以在火焰表中查看此结果的详细信息。

放大时间轴的部分

你可以放大某个记录的一部分，使得分析更佳容易；你可以使用概要窗口放大记录的一部分，在火焰表中会自动匹配相同的部分。

你可以在概要时间轴中用鼠标拖动；
也可以是实用鼠标拖动灰色的条。

以前你可以通过键盘的W,S,A,D来进行操作。

保存和载入记录

你可以打开概要或者火焰表，右键单击选择相关选项。
