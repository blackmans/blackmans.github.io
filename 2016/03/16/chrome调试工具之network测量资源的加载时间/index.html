chrome调试工具之Network测量资源的加载时间（翻译）

  • chrome
  • network
  • 翻译

实用Network面板可以测量你的站点网络加载情况。

Network面板记录了页面中关于每条网络请求的信息，包括时间数据，HTTP请求和响应头信息，cookie等等更多。

Network面板的概览

Networkyou下面的五个面板组成：

时间轴工具有四个面板组成

1.控制面板：使用这些选项来控制Network面板的外观和功能。
2.过滤面板：使用这些选项可以控制展示在请求列表中的请求。小提示：键盘按下Command键，并且点击过滤选项，在同一次过滤中使用多个过滤条件。
3.概要面板：这里的图形代表资源取回后的时间表，如果你看到有些彩色条是垂直堆叠分布的，这意味着这些资源是被同时请求回来的。
4.请求列表面板：这里列出了所有被请求回来的资源。默认这些资源是按照时间顺序排列的，最早的排列在最上面。点击一个资源的名称区域以获取关于此资源的信息。小提示：点击任何列表的头部，都可以添加新的列或者删除任何一列，但是出去timeline不能删除。
5.摘要面板：这里一目了然的显示了当前列表的数量，总数量，加载所需的时间。

请求列表默认展示了一下列，当然你可以自己添加或删除

  • Name 资源的名称
  • Status HTTP状态码
  • Type 请求资源的类型
  • Initiator 发起资源请求的对象或者进程，可能是下列的某一个：
    1.Parser：chrome的解析器发起的请求。
    2.Redirect：一个HTTP重定向发起的请求。
    3.Script：script发起的请求。
    4.Other：一些其他的进程或者活动发起的请求，如用户通过连接导航到页面，或者用户直接在地址栏输入地址。
  • Size：资源的大小，包括响应头（通常几百个字节），再加上响应主题的大小。
  • Time：包括从请求开始响应到最后一个字节下载完毕的时间。
  • Timeline：时间轴通过可视化瀑布流图展示所有请求数据，点击每一栏的头部区域可以按照此列来对数据进行排序。

录制网络请求

默认情况下，在Network面板打开的时候，开发者工具会记录所有的网络活动。

你能通过录制按钮告诉开发者工具是否录制；当按钮是红色的时候表示正在录制中，当按钮是灰色的时候表示没有在录制。你可以通过按钮控制也可以通过快捷键控制Command+e。

查看DOMContentLoaded和Load事件信息

Network面板标注了这两个事件。

DOMContentLoaded是在页面初始节点解析完成时触发，在Network面板中有两个地方显示此事件：
1.在概览面板中蓝色的垂直线就代表此事件。
2.在底部摘要面板中也可以找到蓝色的文字标注。

Load事件触发是在页面完全加载完成的时候触发，在Network中三个地方显示此事件：
1.在概要面板中红色的垂直线代表此事件。
2.在请求列表面板中也是红色的垂直线代表。
3.在摘要面板中红色的文字标注。

查看单个资源的详情

点击某一资源的名称（Name下面的部分）来查看关于此资源的更多信息。

选项卡的个数是由你选择资源的类型决定的，但是以下四个是通用的：

Headers. HTTP headers associated to the resource.
Preview. Previews of JSON, image, and text resources.
Response. HTTP response data (if any).
Timing. A granular breakdown of the request lifecycle for the resource.

  • Headers：资源HTTP请求头的相关信息
  • 预览JSON，image，文本资源
  • HTTP相应数据（如果有的话）
  • 该资源请求生命周期的详细分析

查看Network的timing

点击timing查看单个资源请求生命周期的详细分析。

展示了生命周期内以下类别花费时间的多少：

  • Queuing
  • Stalled
  • Request/Response
  • Request sent
  • Waiting (Time to first byte (TTFB))
  • Content Download

你在请求列表面板中，将鼠标移动到Timeline选项上，也能看到同样的数据。

查看HTTP头

点击headers展示此资源的头部信息，这里展示了此资源的请求URL，HTTP请求方式，HTTP状态码；另外，还列出了请求和响应头，和他们的值以及请求参数。

你也能通过view source和view parsed链接，查看你想要格式。

预览一个资源

点击Preview查看资源的预览，Preview可能展示有用或者没用的信息，这取决于你的选择的资源。

查看HTTP response内容

点击Response查看资源的预览，Response可能展示有用或者没用的信息，这取决于你的选择的资源。

查看cookie

点击cookie查看cookie，这里只有在请求过程中有cookie传播时，才会显示。

下面是一些字段的描述：

  • Name. 名称
  • Value. 值
  • Domain. cookie所属的域（作者：只要在设置域名下的域名都可以访问此cookie）
  • Path. cookie所在的路径（作者：只有此路径或者此路径下一级别的链接能访问此cookie）
  • Expires / Max-Age. 设置cookie的过期时间（作者：前者未来的一个时间点，后者是在给定的时间段后）
  • Size. 大小
  • HTTP. Indicates that the cookie should only be set by the browser in the HTTP request, and cannot be accessed with JavaScript.
  • Secure. 只能在安全连接（作者：指HTTPS）上传输

请求列表的排序

默认请求列表是按照请求开始的时间先后排序的，最先开始的排在最前面。点击每一项的头部记忆按照此项排序，再次点击倒序。

时间轴选项和其他选项是不一样的，当你点击时，它出现一个排序用的下拉菜单选项：

  • Timeline 请求的开始时间，这也是默认的排序方式，和Start Time是一样的。
  • Start Time 和默认是一样的。
  • Response Time 按照请求的响应时间排序。
  • End Time 按照请求的完成时间排序。
  • Duration 按照加载资源总时间排序，可以方便查看那个请求花费时间最长
  • Latency 按照请求开始的到响应开始之间的时间排序，方便查看第一个字节返回花费时间最长的资源。

请求列表过滤

Network面板提供了多重过滤来展示资源。点击左上角的第四个按钮显示或隐藏过滤面板。点击了某一个过滤按钮就会只显示你过滤的资源。

使用Command（Mac）或者ctrl（Windows）点击多选过滤条件。

过滤输入框区域是一个强大的模糊过滤，你如果随便如果一些字符串，他会去匹配资源的名称。
这里也支持其他的方式过滤，按照不同的属性过滤资源，例如使用larger than关键字。

  • domain. 仅仅显示此域下的资源，你可以使用（）来代表多个域，如：.com代表只展示来自*.com域下。开发者工具提供了自动补全的下拉菜单。
  • has-response-header. 展示响应头包含特定内容的资源，开发者工具同样会自动补全。
  • is. Use is:running to find WebSocket resources.
  • larger-than. 展示大于指定指的资源，1000代表时1k。
  • method. 展示对应请求方式的资源，同样会自动补全。
  • mime-type. 展示对应文件MIME类型的资源，同源不全。
  • mixed-content. Show all mixed content resources (mixed-content:all) or just the ones that are currently displayed (mixed-content:displayed).
  • scheme. 展示制定scheme的资源。
  • set-cookie-domain. 展示设置指定domain属性的资源，补全。
  • set-cookie-name. 展示设置对应name的资源，补全。
  • set-cookie-value. 展示设置对应值的资源，补全。
  • status-code. 展示设置对应状态码的资源，补全。

触发补全需要加上后面的冒号，如：domain:。

复制，保存，清除网络信息

在每一条资源上右键单击即可出线菜单选项。这里没有展开翻译。

自定义Network面板

默认请求列表展示的是小行（作者：就是高度小，信息少）。你可以通过Use large resource rows来设置成高行。

添加和删除栏目。在栏目头部右键出现菜单，删除或添加。

默认上次的请求纪录是被丢弃的，你可以勾选Preserve log来保留。
